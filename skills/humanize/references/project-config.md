# 프로젝트 설정 규약 (.yoonmoon.md)

yoonmoon 스킬(humanize·translate-polish·proofread·restyle·detect·polish-all)이 **프로젝트별 단어
사전·규칙**을 읽어 작업에 반영하기 위한 공유 규약이다. 소비 프로젝트가 루트에 `.yoonmoon.md`를 두면,
각 스킬이 Phase 0에서 이 파일을 찾아 읽고 이번 작업의 **우선 기준**으로 삼는다.

스킬은 실행 코드 없이 LLM이 읽어 적용하는 마크다운 지시문이므로, 설정 파일도 LLM이 그대로 읽어
적용하는 마크다운이다. 별도 파서가 필요 없다.

---

## 1. 파일 위치와 탐색

- 파일명: **`.yoonmoon.md`** (프로젝트 루트 권장).
- **탐색 순서**: 현재 작업 디렉터리에서 시작해 **상위로 올라가며** 첫 번째 `.yoonmoon.md`를 찾는다
  (git 동작과 유사). **가장 가까운(하위) 파일이 우선**이며, 발견 즉시 멈춘다. git 저장소 루트 또는
  파일시스템 루트에 닿으면 탐색 종료.
- 여러 위치에 파일이 있어도 **병합하지 않는다** — 가장 가까운 하나만 쓴다(결정적 동작).
- 탐색 예시(셸):

  ```bash
  d="$PWD"; while [ "$d" != "/" ]; do
    [ -f "$d/.yoonmoon.md" ] && { echo "$d/.yoonmoon.md"; break; }
    d="$(dirname "$d")"
  done
  ```

- 파일이 없으면 **설정 없이** 스킬 기본값으로 진행한다(이 기능은 전적으로 선택적이며, 없을 때 동작이
  바뀌면 안 된다).
- 대화에 붙여넣은 텍스트만 다루는 등 프로젝트 컨텍스트가 없으면 탐색을 생략해도 된다. 사용자가
  설정 경로를 직접 지정하면 그 파일을 쓴다.

---

## 2. 섹션 (모두 선택)

`.yoonmoon.md`는 아래 마크다운 헤딩 섹션으로 구성한다. **있는 섹션만** 적용하고 없으면 무시한다.
헤딩 문구는 아래 표준명을 권장하되, 의미가 통하면 유연하게 해석한다(예: "금지어"="금칙어").

| 섹션 | 헤딩(권장) | 역할 |
| ---- | ---------- | ---- |
| 보존 사전 | `## 보존 사전` | 절대 바꾸지 않을 표현(브랜드·제품명·고유 용어). 윤문·교정·변환 모두에서 원형 유지. detect는 AI 신호로 세지 않음. |
| 치환 사전 | `## 치환 사전` | `A → B` 강제 치환. 본문에 A가 나오면 의미를 해치지 않는 선에서 B로 바꾼다. |
| 금칙어 | `## 금칙어` | 피해야 할 표현. 윤문·번역투 제거에서 우선 제거 대상으로 가중. |
| 선호 표현 | `## 선호 표현` | 같은 뜻의 후보 중 우선할 표현(소프트 선호). 강제는 아님. |
| 기본 옵션 | `## 기본 옵션` | genre / intensity(강도) / register(종결체) / source(원천 언어) 등 기본값. |
| 예외 규칙 | `## 예외 규칙` | 자유 산문 규칙. 특정 AI-tell/번역투 카테고리 끄기·허용, 조직 스타일 지침 등. |

각 섹션 작성 형식:

- **보존 사전 / 금칙어 / 선호 표현**: 불릿 목록. 항목 뒤 괄호로 주석 가능 — 예: `- onboarding (번역 금지)`.
- **치환 사전**: 한 줄에 하나씩 `잘못된말 → 바른말`. 방향이 분명해야 한다.
- **기본 옵션**: `키: 값` 한 줄씩. 키는 `genre`·`intensity`·`register`·`source` 등 스킬이 Phase 0에서
  파싱하는 옵션명과 맞춘다.
- **예외 규칙**: 자연어 문장/불릿. 예: "블로그 글이므로 이모지는 약한 신호로 본다", "카테고리 10(영어
  인용)은 검사하지 않는다".

---

## 3. 우선순위

충돌 시 **위가 강하다**:

1. **사용자가 이번 대화에서 직접 지정한 옵션·지시** (가장 강함)
2. **`.yoonmoon.md` 프로젝트 설정**
3. **스킬 기본값**

사전 간 충돌은 **보존 사전이 최우선**이다. 어떤 표현이 보존 사전에 있으면 치환 사전·금칙어보다 우선해
원형을 지킨다(브랜드명이 우연히 금칙 패턴과 겹쳐도 건드리지 않는다).

---

## 4. 스킬별 적용

- **humanize / translate-polish** — 보존 사전을 절대 보존 목록에 추가, 치환 사전을 강제 edit으로,
  금칙어를 제거 가중치로, 기본 옵션을 Phase 0 옵션 기본값으로, 예외 규칙으로 카테고리 심각도를 조정.
  자체검증 시 치환 누락·보존 위반·금칙어 잔존을 추가 점검 항목으로 본다.
- **proofread** — 보존 사전(고유 표기 유지)·치환 사전(조직 표준 표기 강제)·기본 옵션(표기 기준)을
  반영. 규범 교정 범위는 유지하되, 보존 사전 항목은 비표준이어도 교정하지 않는다(의도된 표기로 간주).
- **restyle** — 기본 옵션의 register를 목표 문체 기본값으로, 보존 사전·치환 사전을 변환 중 유지/적용.
- **detect** — 보존 사전 항목과 예외 규칙으로 **위양성을 억제**(조직 표준 표현을 AI 신호로 세지 않음).
  기본 옵션의 genre를 장르 추정 기본값으로. 글을 고치지 않으므로 치환·금칙어는 적용하지 않는다.
- **polish-all** — Phase 0에서 설정을 한 번 로드해 한 줄 상태에 반영하고, 각 하위 스킬이 스스로
  같은 파일을 로드하므로 별도 전달은 불필요. 설정 적용 여부를 최종 보고에 한 줄로 명시.

설정을 적용했으면 작업 시작 한 줄 상태에 `설정: .yoonmoon.md 적용`처럼 표시한다(없으면 표시 안 함).

## 5. 템플릿

```markdown
# .yoonmoon.md — yoonmoon 프로젝트 설정

## 보존 사전
- 다이어트친구
- DietFriends
- onboarding (번역 금지)

## 치환 사전
유저 → 사용자
어플 → 앱
딥러닝 모델 → 딥러닝 모델

## 금칙어
- ~에 대해
- ~을 통해
- 매우, 굉장히

## 선호 표현
- 그러나 (← 하지만)

## 기본 옵션
genre: 공식문서
intensity: 기본
register: 해요체

## 예외 규칙
- 블로그 글에서는 이모지를 약한 신호로 본다.
- 카테고리 10(영어 인용 과다)은 검사하지 않는다 — 기술 용어 원어 병기를 허용한다.
```
