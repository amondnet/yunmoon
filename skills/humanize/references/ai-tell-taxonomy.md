# 윤문 AI 티 분류 (humanize AI-tell taxonomy)

AI(ChatGPT·Claude·Gemini 등)가 쓴 한글 텍스트에서 반복적으로 나타나는 "AI 티"를 10대 카테고리로
정리한 탐지 기준이다. 각 항목은 **정의 · 패턴/예시 · 심각도 · 보존 주의**로 구성된다.

## 심각도 등급

| 등급   | 의미                                           | 처리                     |
| ------ | ---------------------------------------------- | ------------------------ |
| **강** | 한 번만 나와도 AI 티가 뚜렷함                  | 원칙적으로 모두 제거     |
| **중** | 1~2회는 자연스러우나 반복되면 티가 남          | 문서당 3회 이상이면 제거 |
| **약** | 단독으로는 문제없고 다른 패턴과 겹칠 때만 부각 | 중첩 시에만 손봄         |

심각도는 **장르·문맥에 따라 조정**한다. 예: 불릿·이모지는 블로그에서는 약, 공식문서·리포트에서는 강.

## 1. 번역투

영어·일본어 구문을 그대로 옮긴 듯한 어색한 구조. AI 글의 가장 흔한 티.

| 패턴                            | 예시 (티 → 자연)                                                  | 심각도 |
| ------------------------------- | ----------------------------------------------------------------- | ------ |
| `~에 대해/~에 관해` 남용        | 환경 문제에 대해 논의한다 → 환경 문제를 논의한다                  | 중     |
| `~을 통해/~를 통하여` 경로 도식 | 교육을 통해 성장한다 → 교육으로 성장한다 / 교육받으며 성장한다    | 중     |
| 무생물 주어 + 타동사            | 이 연구는 ~을 보여준다 → 이 연구로 ~을 알 수 있다                 | 중     |
| 직역 대명사 `그/그녀/그것/그들` | 그것은 중요하다. 그들은… → (반복 대명사 생략 또는 지시 대상 명시) | 강     |
| `가지다/갖다` 직역(have)        | 중요성을 가진다 → 중요하다                                        | 중     |
| `~중 하나` (one of the)         | 가장 중요한 요소 중 하나 → 특히 중요한 요소                       | 약     |
| 무생물 복수 `-들`               | 정보들·문제들·결과들 → 정보·문제·결과                             | 중     |
| 우향 분기 관계절 중첩           | ~하는, ~인, ~한 X → 절을 끊어 문장 분할                           | 중     |

> **근거:** 한국 번역학은 직역 대명사·무생물 복수 '-들'·무생물 주어를 영한 번역투의 핵심 유형으로 분류한다
> (김도훈 2009; 이영옥 2001; 최서영 2022). 이론적으로는 Toury(1995)의 간섭 법칙·Baker(1996)의 명시화로 설명되며,
> Toral(2019)은 MT/AI 산출물에서 이 번역투가 오히려 *악화*됨을 보였다. **관형절 중첩·긴 전치수식**도 영한 번역문의
> 대표 문제로, 영어 어순 모사·관형격 '의' 반복·다중 내포가 용인성·가독성을 떨어뜨린다(김혜영 2011; 성승은 2011 —
> 번역서가 비번역서보다 관형절이 더 김). 단 좌분지는 한국어의 **정상** 특징이므로 '개수·길이'가 아니라 **번역투
> 구조**일 때만 손본다(과교정 금지) → [`translationese-research.md`](../../translate-polish/references/translationese-research.md).

**주의:** 직역 대명사라도 인물 인용·대화에서 꼭 필요한 경우는 보존. "그"가 특정 인물을 명확히 가리키며
생략 시 모호해지면 유지.

## 2. 피동·사동 남용

영어 수동태의 잔재. 능동으로 바꾸면 대부분 또렷해진다.

| 패턴                             | 예시 (티 → 자연)                                                                             | 심각도 |
| -------------------------------- | -------------------------------------------------------------------------------------------- | ------ |
| 이중 피동                        | 되어진다 / 보여진다 / 생각되어진다 / 이루어지게 된다 → 된다 / 보인다 / 생각된다 / 이루어진다 | 강     |
| `~에 의해` 수동                  | 전문가에 의해 작성되었다 → 전문가가 작성했다                                                 | 중     |
| 불필요한 `~로 보여진다/여겨진다` | 효과적으로 보여진다 → 효과적이다 / 효과적으로 보인다                                         | 중     |
| 상태 피동 남발 `~되어 있다`      | 구성되어 있다(반복) → 구성된다 / ~이다                                                       | 약     |

> **근거:** 피동·이중피동 남용은 일한·영한 번역투의 대표 유형으로 보고된다(오경순 2010; 최유숙 2023).
> 영-한 MT 품질평가는 이런 부자연성을 fluency 오류("Unnaturalness")로 코딩한다(Park & Padó, LREC-COLING 2024)
> → [`translationese-research.md`](../../translate-polish/references/translationese-research.md).

**주의:** 행위자가 불분명하거나 의도적으로 감추는 공식·법률 문체의 피동은 보존 가능
(예: "본 약관은 다음과 같이 정한다"는 그대로).

## 3. 상투적 표현·hedging

내용 없이 자리만 채우는 정형구와 과도한 완충 표현, 홍보성 과장어.

| 패턴                    | 예시 (티 → 자연)                                                                         | 심각도 |
| ----------------------- | ---------------------------------------------------------------------------------------- | ------ |
| 정형 결론 도입          | 결론적으로 / 종합하면 / 요컨대 / 정리하자면 → (삭제하거나 본론에 녹임)                   | 중     |
| 메타 진입구             | ~라는 점에서 주목할 만하다 / 흥미로운 점은 ~이다 / 여기서 중요한 것은 → (바로 본론 진술) | 중     |
| hedging 중첩            | ~라고 할 수 있다 / ~인 것으로 보인다 / ~일 수 있다(연쇄) → 단정 또는 1회로 축약          | 중     |
| 홍보·과장 어휘          | 혁신적인 / 획기적인 / 강력한 / 놀라운 / 필수적인 → 구체적 서술로 대체                    | 강     |
| `중요한 역할을 한다` 류 | ~에서 중요한 역할을 한다 → ~을 좌우한다 / 구체 동사                                      | 중     |
| 균형 강박               | 장점과 단점이 모두 존재한다 / 한편으로는~다른 한편으로는 → 핵심만 진술                   | 약     |

**주의:** 학술·리포트에서 근거가 제한적일 때의 정당한 hedging(예: "예비 결과로는")은 보존.
모든 단정을 강제로 만들지 말 것 — 의미가 바뀐다.

## 4. 접속사·연결어 남발

문장마다 접속사로 시작하거나 연결어미 뒤에 쉼표를 찍는 기계적 습관.

| 패턴                             | 예시 (티 → 자연)                                                      | 심각도 |
| -------------------------------- | --------------------------------------------------------------------- | ------ |
| 연결어미 뒤 쉼표                 | 그리고, ~ / 하지만, ~ / 또한, ~ → 쉼표 제거 또는 접속사 자체 삭제     | 강     |
| 문두 접속사 반복                 | 모든 문장이 또한/따라서/그러므로로 시작 → 일부만 남기고 흐름으로 연결 | 중     |
| `즉` 재정의 남발                 | 즉, ~. 즉, ~ → 1회로 제한 또는 삭제                                   | 중     |
| `뿐만 아니라/더 나아가/무엇보다` | (반복 시) → 빈도 축소                                                 | 약     |

> **근거:** 한국어는 영어보다 쉼표를 덜 쓰는데 LLM은 연결어미·접속부사 뒤에 쉼표를 과용한다.
> 쉼표 기반 특징은 KatFishNet(ACL 2025)에서 가장 강력한 LLM 판별자였다(에세이 AUC-ROC 94.88%)
> → [`katfishnet-research.md`](katfishnet-research.md).

**주의:** 논리 전환이 꼭 필요한 지점의 접속사(역접 "하지만", 인과 "따라서")는 보존.
접속사를 0으로 만들면 오히려 비약이 생긴다.

## 5. 기계적 구조·병렬

내용과 무관하게 형식이 지나치게 규칙적이어서 기계가 찍어낸 인상을 주는 경우.

| 패턴                       | 예시 (티 → 자연)                                          | 심각도 |
| -------------------------- | --------------------------------------------------------- | ------ |
| `첫째/둘째/셋째` 정형 나열 | (짧은 글에서) → 산문으로 자연스럽게 연결                  | 중     |
| 불릿 과다                  | 모든 항목을 •로 나열 → 설명형 산문 전환(또는 핵심만 불릿) | 중     |
| 정형 헤더 남발             | ## 서론 / ## 본론 / ## 결론 → 내용 기반 제목 또는 제거    | 약     |
| 균일 문장 길이·구조        | 비슷한 길이의 평서문 연속 → 장단 변주                     | 중     |
| `X에서 Y로` 변환 도식      | 단순한 도구에서 필수 파트너로 → 구체 진술                 | 약     |

**주의:** 절차·체크리스트·사양처럼 **불릿/번호가 본질적으로 적합한 장르**는 유지.
구조 자체가 정보인 표·목록을 산문으로 풀지 말 것.

## 6. 형식명사·명사화

동사로 쓰면 될 것을 명사로 굳혀 문장을 늘어지게 만드는 습관.

| 패턴                      | 예시 (티 → 자연)                                            | 심각도 |
| ------------------------- | ----------------------------------------------------------- | ------ |
| `것이다` 종결 남발        | ~할 것이다 / ~인 것이다(반복) → ~한다 / ~이다               | 중     |
| `~하는 것` 명사화         | 이해하는 것이 중요하다 → 이해가 중요하다 / 이해해야 한다    | 중     |
| `~한다는 점/~라는 점에서` | 효율적이라는 점에서 → 효율적이어서                          | 중     |
| 한자어 명사화 + 하다      | 활용을 하다 → 활용하다 / ~의 구현·제공·활용 → 동사화        | 중     |
| 의존명사 체인             | ~할 필요가 있다 / ~할 수 있는 바 → ~해야 한다 / ~할 수 있다 | 중     |

> **근거:** 한국어 명사화는 체계기능언어학의 **문법적 은유**로, 학술·격식 텍스트의 정보밀도·객관적 문체를 만드는
> 어휘문법 자원이다(정려란 2018; 소지영·주세형 2017 — 명사화는 고학년·학술 텍스트일수록 증가). 학술 문어가 절 확장이
> 아니라 **명사구 압축**으로 복잡해진다는 영어권 정설과도 합치한다(Biber & Gray 2010). 따라서 명사화 과잉은
> "AI=격식·문어체 편향"의 표지다. 단 **경동사 분리('활용을 하다'·한자어 대동사 '실시·수행·진행하다')를 번역투/AI 티로 규정한
> 학술 1차 문헌은 미발굴**(2026-06 재조사 확인) → 홍성호(2020, 한국경제 '서술어 3종 세트')·이오덕·이수열
> 글쓰기서·국립국어원 「쉬운 공문서 쓰기 길잡이」 등 **군더더기 간결화** 처방을 따른다.
> [`translationese-research.md`](../../translate-polish/references/translationese-research.md) 참조.

**주의:** 의미상 명사화가 더 정확한 전문·법률 표현은 보존. 무조건 동사화하면 어색해질 수 있다.

## 7. 과잉 수식

강조를 남발해 오히려 힘이 빠지는 경우. 군더더기 부사·접미사·동의어 중첩.

| 패턴          | 예시 (티 → 자연)                                                             | 심각도 |
| ------------- | ---------------------------------------------------------------------------- | ------ |
| 강조부사 남발 | 매우 / 정말 / 너무 / 굉장히 / 대단히(빈발) → 대부분 삭제                     | 중     |
| `-적` 연쇄    | 기술적 토대 / 사회적 측면 / 효율적 방식 → 기술 기반 / 사회 측면 / 효율적으로 | 중     |
| 동의어 중첩   | 도움과 지원을 / 빠르고 신속하게 → 하나로 축약                                | 약     |
| 군더더기 부사 | 기본적으로 / 사실상 / 본질적으로(의미 없는 자리) → 삭제                      | 약     |

**주의:** 정도를 정확히 전달해야 하는 곳의 부사(예: "거의 두 배")는 보존. 수치·정도 부사는 함부로 지우지 말 것.

## 8. 시각 장식

마크다운·이모지·대시로 글을 과하게 꾸며 기계 출력처럼 보이게 하는 경우.

| 패턴                   | 예시 (티 → 자연)                                 | 심각도                       |
| ---------------------- | ------------------------------------------------ | ---------------------------- |
| 과도한 볼드            | 문장 곳곳 **강조** → 정말 핵심 1~2곳만 또는 제거 | 중                           |
| 이모지                 | 🚀 ✨ 🔥 등 본문 삽입 → 제거(블로그는 일부 허용) | 강(공식·리포트) / 약(블로그) |
| em-dash(—) 영어식 남용 | A — B → 쉼표·괄호·연결어로 전환                  | 중                           |
| 따옴표 강조 남용       | 불필요한 '강조' 따옴표 → 제거                    | 약                           |

**주의:** 코드·표·실제 인용 부호·정식 명칭의 따옴표는 보존. 마크다운이 요구되는 문서(README 등)의
구조적 강조는 장르에 맞게 유지.

---

## 9. 리듬 균일성

문장의 길이와 종결, 문두가 단조롭게 반복돼 사람 글 특유의 리듬이 없는 경우.

| 패턴           | 예시 (티 → 자연)                                            | 심각도 |
| -------------- | ----------------------------------------------------------- | ------ |
| 종결어미 단조  | ~이다.~이다.~이다. → ~다 / ~ㄴ다 / 명사 종결 / 의문 등 변주 | 중     |
| 문장 길이 균일 | 비슷한 길이 연속 → 짧은 문장과 긴 문장 교차                 | 중     |
| 동일 문두 반복 | 이러한~ / 이는~ / 이를 통해~ 반복 → 문두 다양화             | 중     |

> **근거:** 사람 글은 품사 n-gram(통사 구조)이 다양한 반면 LLM은 흔한 구조를 반복한다.
> POS 다양성 저하는 KatFishNet(ACL 2025)의 핵심 판별 특징 중 하나다 → [`katfishnet-research.md`](katfishnet-research.md).

**주의:** 리듬 변주는 **의미를 바꾸지 않는 선**에서만. 격식체(하십시오체/해요체) 일관성은 유지하고
종결을 섞느라 register를 흩뜨리지 말 것.

## 10. 영어 인용 과다

한국어로 충분한데 영어를 병기하거나 번역하지 않고 남겨 번역기 인상을 주는 경우.

| 패턴               | 예시 (티 → 자연)                                                             | 심각도 |
| ------------------ | ---------------------------------------------------------------------------- | ------ |
