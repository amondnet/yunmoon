# 윤문 풀코스 파이프라인 가이드 (polish-all pipeline guide)

한 편의 한글 글을 **진단 → 교정 → 번역투 제거 → AI 티 제거 → 문체 변환 → 재진단**의 순서로 한 흐름에서
처리하는 오케스트레이터의 기준이다. 각 단계는 형제 스킬의 reference를 그대로 재사용하며, 이 가이드는
**단계 순서·라우팅·중복 처리·통합 검증**만 정의한다(개별 규칙은 각 스킬 reference 참조).

## 핵심 불변식

1. **의미 보존(restyle 제외)** — proofread·translate-polish·humanize는 내용을 한 글자도 바꾸지 않는다.
   restyle만 register를 **의도적으로** 바꾼다(그래서 맨 마지막, 그리고 목표 지정 시에만).
2. **누적 적용** — 각 단계의 출력이 다음 단계의 입력이 된다. 앞 단계가 만든 결과를 뒤 단계가 존중한다.
3. **단계 건너뛰기는 명시** — 라우팅으로 생략한 단계는 보고서에 `skip(사유)`로 남긴다(조용히 빠뜨리지 않는다).
4. **과교정 금지 누적** — 단계가 쌓일수록 과교정 위험이 커진다. 각 단계는 자기 범위만 손대고, 앞 단계가
   이미 고친 부분을 되돌리거나 다시 건드리지 않는다.

## 단계 순서와 근거

| 순서 | 단계         | 스킬 reference                                                    | 왜 이 위치인가                                  |
| ---- | ------------ | ----------------------------------------------------------------- | ----------------------------------------------- |
| 1    | 진단·라우팅  | `../../detect/references/lread-rubric.md` 등                      | 어느 cleaner를 켤지 먼저 정해야 낭비가 없다     |
| 2    | 교정·교열    | `../../proofread/references/proofreading-rules.md`                | 규범 오류를 먼저 없애 깨끗한 토대를 만든다      |
| 3    | 번역투 제거  | `../../translate-polish/references/translationese-taxonomy.md` 외 | 구조적 번역투를 먼저 풀면 AI 티 작업이 쉬워진다 |
| 4    | AI 티 제거   | `../../humanize/references/ai-tell-taxonomy.md` 외                | 번역투 정리 후 남은 AI 고유 티(상투어·리듬 등)  |
| 5    | 문체·톤 변환 | `../../restyle/references/register-guide.md`                      | register 변경은 의미 보존 단계가 끝난 뒤 마지막 |
| 6    | 재진단(QA)   | detect reference 재사용                                           | 다듬은 결과에 AI 티·번역체가 남았는지 확인      |

> 2~4는 모두 의미 보존 단계라 순서를 바꿔도 결과 의미는 같지만, **교정→번역투→AI티** 순이 과교정을 가장
> 줄인다(오류 제거 → 구조 정리 → 표면 다듬기). 5는 의미(=register)를 바꾸므로 반드시 마지막.

## 라우팅 매트릭스

**Phase 1(진단)** 결과로 어느 단계를 켤지 정한다. `proofread`는 항상, `restyle`은 목표 문체 지정 시에만.

| 진단 결과                  | proofread | translate-polish | humanize        | restyle      |
| -------------------------- | --------- | ---------------- | --------------- | ------------ |
| 사람 원작 (AI✗·번역✗)      | ✅        | ⏭ skip          | ⏭ skip         | 목표 지정 시 |
| 번역문 (AI✗)               | ✅        | ✅               | ⏭ skip         | 목표 지정 시 |
| AI 글 (번역✗)              | ✅        | ⏭ skip          | ✅              | 목표 지정 시 |
| AI가 번역한 글 (AI✓·번역✓) | ✅        | ✅               | ✅              | 목표 지정 시 |
| 불확실                     | ✅        | 번역투 보이면 ✅ | AI 티 보이면 ✅ | 목표 지정 시 |

- 사용자가 단계를 명시(예: "번역투는 빼고")하면 라우팅보다 **사용자 지정이 우선**.
- "AI인지 모르겠지만 그냥 다 돌려줘" → 불확실 행을 따르되, 패턴이 없으면 해당 cleaner는 변경 0건으로 통과.

## translate-polish ↔ humanize 중복 처리

두 단계가 모두 켜지면(AI가 번역한 글) 패턴이 겹친다(번역투=AI 티 카테고리 1·2·6). **이중 편집을 피한다.**

- **3단계(translate-polish)**: 번역투 8유형(무생물 주어·직역 대명사·피동·조사·명사화 등)을 **전담 처리**한다.
- **4단계(humanize)**: 번역투는 이미 처리됐다고 보고, **AI 고유 티에 집중**한다 — 상투어·hedging(3), 접속사
  남발(4), 기계적 병렬·불릿(5), 과잉 수식(7), 시각 장식(8), 리듬 균일성(9), 영어 인용(10).
- 4단계에서 번역투 잔존이 보이면 그때만 보강한다(되돌리지 않는다).

## 강도·옵션 전달

- **강도(`보수|기본|적극`)**: 파이프라인 옵션을 각 cleaner(proofread 제외 — 교정은 강도 무관)에 그대로 전달한다.
- **장르**: Phase 1에서 추정한 장르를 humanize·translate-polish의 심각도 조정에 공유한다.
- **목표 문체**: restyle용. 지정 없으면 5단계 skip.
- **원문(원천 텍스트)**: 있으면 translate-polish의 명시화·간섭 대조에 전달한다.

## 통합 자체검증

각 단계는 자기 스킬의 자체검증을 수행한다. 파이프라인 종료 시 **전체 누적 검증**을 한 번 더 한다.

1. **사실 보존** — 최초 입력 대비 고유명사·수치·인용·날짜·단위가 100% 그대로인가?(restyle의 register 변경은 예외)
2. **의미 동등** — 주장 방향·인과·수량·강도가 최초 입력과 같은가?
3. **단계 간 충돌 없음** — 뒤 단계가 앞 단계의 정당한 수정을 되돌리지 않았는가?
4. **과교정 누적 점검** — 누적 변경률이 과한가?(대략 50% 초과면 보수적으로 재검토)
5. **최종 일관성** — 종결체·격식·표기가 글 전체에서 일관되는가?
6. **재진단 개선** — Phase 6 detect 재진단에서 AI 가능성·번역체 신호가 입력 대비 낮아졌는가?

위반 시 해당 단계로 돌아가 1회 부분 재작업한다.

## 출력 형식

1. **한 줄 상태** — `완료. 실행 단계: {켜진 단계} / 누적 변경률 X% / 재진단: AI 가능성 {전}→{후}`
2. **최종 결과물** — 모든 단계를 거친 글(코드블록/인용).
3. **단계별 요약 표**

   | 단계        | 실행                | 주요 변경/탐지        |
   | ----------- | ------------------- | --------------------- |
   | 진단·라우팅 | AI 글로 판정        | humanize+proofread on |
   | 교정        | ✅ 3건              | 맞춤법 2 · 띄어쓰기 1 |
   | 번역투      | ⏭ skip(번역 아님)  | —                     |
   | AI 티       | ✅ 12건             | 상투어 4 · 리듬 …     |
   | 문체 변환   | ⏭ skip(목표 없음)  | —                     |
   | 재진단(QA)  | AI 가능성 높음→낮음 | 개선 확인             |

4. **before → after 하이라이트 3~5건**
5. (있으면) **확인 요망** — 오역 의심·효력 문구·needs_discussion 항목.
