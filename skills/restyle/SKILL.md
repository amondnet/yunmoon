# 문체·톤 변환(restyle) — 격식·매체 맞춤

<role>
한국어 글의 **내용은 보존하면서 문체·격식·톤을 목표에 맞게 바꾸는** 도구다. 윤문(humanize)이 *문체를
유지한 채* AI 티만 지운다면, 이 스킬은 *문체 자체를 의도적으로 바꾼다*. 탐지(현재 문체 파악) → 변환 →
검증을 **이 컨텍스트 안에서 한 흐름으로** 처리한다(별도 서브에이전트를 띄우지 않는다).
</role>

<resources>
필요한 단계에서 아래 파일을 읽어 기준으로 삼는다.

- `../humanize/references/project-config.md` — 프로젝트별 단어 사전·규칙(`.yoonmoon.md`) 규약. **Phase 0**에서 설정 파일을 찾으면 읽는다.
- `references/register-guide.md` — 상대높임 종결 체계, 문어↔구어 특징, 매체·장르별 톤, 변환 일관성 규칙, 보존 원칙. **Phase 1~3** 전반의 기준.
  </resources>

<workflow>

<phase n="0" name="목표 문체 확인">
1. 변환할 한국어 텍스트를 확보한다(붙여넣기 또는 파일).
2. **목표(target)**를 파싱한다 — 종결체(하십시오체/해요체/한다체/반말), 격식(격식↔캐주얼), 매체
   (보도자료/이메일/CS/마케팅/SNS/블로그/학술 등) 중 사용자가 지정한 것. 모호하면 가장 가까운 한 줄로
   되묻거나, "현재 → 목표"를 명시하고 진행한다.
3. **프로젝트 설정 로드** — 현재 디렉터리에서 상위로 `.yoonmoon.md`를 탐색해 발견하면 읽는다(규약은
   `../humanize/references/project-config.md`). 목표가 모호하면 **기본 옵션의 register**를 목표 기본값으로
   쓰고, 보존 사전·치환 사전을 변환 중 유지·적용한다. 없으면 기본값으로 진행.
4. 작업 시작 시 한 줄 상태를 출력한다: `restyle — 현재: {현재 문체} → 목표: {목표 문체/매체}`{설정 적용 시 ` / 설정: .yoonmoon.md`}
</phase>

<phase n="1" name="현재 문체 파악">
`references/register-guide.md`로 입력의 현재 종결체·격식·문어/구어·어휘 수준·호칭을 파악한다.
변환 폭(예: 한다체→반말은 큰 폭)을 가늠한다.
</phase>

<phase n="2" name="변환">
목표 문체로 바꾼다. **종결만 바꾸지 말고** `register-guide.md`의 일관성 규칙을 함께 적용한다.

- **내용 100% 보존**: 사실·수치·고유명사·인용·주장의 방향과 강도. 톤을 바꾼다고 의미를 바꾸지 않는다.
- **종결 통일** + **높임 '-시-' 정합** + **어휘 격식 맞추기** + **호칭·지칭 일치** + **구어/문어 요소 조정**.
- 매체가 지정되면 해당 톤 규칙(구조·장식 관습)까지 적용한다.
- **프로젝트 설정 반영**(있으면): 보존 사전은 원형 유지, 치환 사전은 강제 적용한다.
- 법적·계약 효력 문구는 격식 변환 시 신중히(의미가 곧 효력 → 필요하면 표시).
  </phase>

<phase n="3" name="검증">
결과물을 원문과 대조해 점검한다: ① 내용·사실 불변 ② 목표 종결체로 **일관** 통일 ③ 높임·호칭 정합
④ 어휘 격식이 종결과 어긋나지 않음 ⑤ 매체 톤 부합. 위반 시 해당 부분을 **부분 재변환**한다.
</phase>

<phase n="4" name="결과 반환">
`<outputFormat>`에 정의한 형식대로 반환한다. 목표가 모호했으면 가정한 목표를 한 줄로 밝히고 조정을 제안한다.
</phase>

</workflow>

<outputFormat>
다음을 순서대로 반환한다.

1. **한 줄 상태** — `완료. {현재 문체} → {목표 문체} / 종결 통일 · 내용 보존`
2. **변환본** — 마크다운 코드블록 또는 인용으로 전체 제시.
3. **변환 요약 표**

   | 항목      | 변환 전   | 변환 후       |
   | --------- | --------- | ------------- |
   | 종결체    | 한다체    | 해요체        |
   | 격식      | 격식·문어 | 비격식·친근   |
   | 호칭/지칭 | (없음)    | 여러분 / 저희 |

4. **주요 변환 하이라이트 3~5건** — `원문 → 변환본` 형태.
5. (있으면) **확인 요망** — 효력 문구 등 신중히 다룬 부분.
   </outputFormat>

<preservationRules>
문체는 바꾸되 어떤 경우에도 보존한다(자세히는 `register-guide.md`의 보존 원칙):
사실·수치·고유명사·날짜·인용 · 주장의 방향과 강도 · 글의 목적·요청·결론 · 법적·계약 효력 문구의 의미 ·
인용·대사 속 원화자의 말투. 의미까지 바뀌면 변환이 아니라 재작성이다(범위 밖).
</preservationRules>

<followupCommands>
| 사용자 신호 | 처리 |
|---|---|
| "더 격식 있게/캐주얼하게" | 목표 격식 수준을 한 단계 조정해 재변환 |
| "종결만 바꿔" | 어휘·구조는 두고 종결체만 통일 |
| "이 매체로" | 지정 매체 톤 규칙으로 재변환 |
| "이 문단만" | 지정 문단만 입력으로 삼아 재실행 |
</followupCommands>

<outOfScope>
- 문체를 유지한 채 AI 티만 제거(윤문) → **humanize** 스킬
- 번역체 교정 → **translate-polish** 스킬
- 맞춤법·띄어쓰기·문법 교정 → **proofread** 스킬
- AI 작성 여부 판별 → **detect** 스킬
- 내용 추가·삭제·요약·재구성을 동반한 재작성 → 별도 글쓰기
</outOfScope>
