# 문체·톤 변환 가이드 (register & tone guide)

글의 **내용은 보존하면서 문체·격식·톤을 의도적으로 바꾸는** 기준이다. 윤문(humanize·translate-polish)이
register를 *유지*한다면, 이 스킬은 register를 _목표에 맞게 바꾼다_. 한국어 상대높임 체계와 문어/구어
특징, 매체별 톤을 정리한다.

## 1. 상대높임 종결 체계

한국어 종결법은 격식/비격식 × 높임 정도로 나뉜다. **목표 문체 하나로 문서 전체를 통일**한다(혼용 금지).

| 종결체         | 격식          | 예 (평서)         | 주 쓰임                                 |
| -------------- | ------------- | ----------------- | --------------------------------------- |
| 하십시오체     | 격식·높임     | 합니다 / 하십니다 | 공지·발표·격식 비즈니스·고객 응대       |
| 해요체         | 비격식·높임   | 해요 / 했어요     | 친근한 안내·블로그·CS·뉴스레터          |
| 하오체·하게체  | 격식·예스러움 | 하오 / 하게       | 문학·번역·예스러운 톤(현대 실무엔 드묾) |
| 한다체(해라체) | 격식·문어     | 한다 / 했다       | 신문·논문·보고서·책(문어 평서의 기본)   |
| 해체(반말)     | 비격식·낮춤   | 해 / 했어         | SNS·친구·캐주얼·내부 메신저             |

> **한다체 vs 하십시오체**: 둘 다 '격식'이지만 한다체는 _독자에게 말 걸지 않는_ 문어 서술이고,
> 하십시오체는 _청자에게 높여 말하는_ 구어·격식 대화체다. 보고서는 한다체, 발표·안내는 하십시오체.

## 2. 문어 ↔ 구어

| 축        | 문어(글말)          | 구어(입말)                          |
| --------- | ------------------- | ----------------------------------- |
| 종결      | 한다체, 명사형 종결 | 해요체·해체, 종결어미 다양          |
| 어휘      | 한자어·전문어 비중↑ | 고유어·일상어, 줄임말·신조어 허용   |
| 문장 길이 | 길고 복합문         | 짧고 끊어짐, 도치·생략              |
| 연결      | 접속부사·관형절     | 그래서·근데 등 구어 접속, 문장 분할 |
| 담화 표지 | 적음                | 응·뭐·좀·진짜 등(과하지 않게)       |
| 호응      | 완결·정형           | 생략·여운 허용                      |

## 3. 매체·장르별 톤

목표 매체가 지정되면 해당 톤 규칙을 적용한다.

- **보도자료/공지** — 하십시오체 또는 한다체, 두괄식, 사실 중심, 군더더기·감정어 배제, 날짜·주체 명확.
- **비즈니스 이메일** — 하십시오체, 인사–용건–요청–맺음 구조, 정중·간결, 모호한 부탁 금지.
- **고객 응대(CS)** — 해요체/하십시오체, 공감→설명→해결, 책임 회피·수동 표현 자제, 따뜻하되 정확히.
- **마케팅/카피** — 짧고 강한 문장, 청자 지칭(여러분·당신), 동사·체언 종결 변주, 과장·허위 금지.
- **SNS/캐주얼** — 해체/해요체, 짧은 호흡, 이모지·줄임말 허용(매체 관습 내), 친근한 말 걸기.
- **블로그/뉴스레터** — 해요체 기본, 적당한 구어, 소제목·예시로 가독성.
- **학술/리포트** — 한다체, 객관·절제, 1인칭·감정어 자제, 정의·근거 명시.
- **격식 서신/공문** — 하십시오체, 정형 인사·상투 격식구 일부 허용, 직함·호칭 정확.

## 4. 변환 시 일관성 규칙

문체만 바꾸고 내용은 그대로 두되, 아래를 함께 맞춰야 자연스럽다.

1. **종결 통일** — 목표 종결체로 문서 전체를 통일. 해요체와 하십시오체, 한다체와 해체를 섞지 않는다.
2. **높임 선어말 '-시-' 정합** — 주체높임이 종결 높임 수준과 어긋나지 않게(반말인데 '-시-' 남발 금지).
3. **어휘 격식 맞추기** — 격식↑이면 한자어·정형어, 캐주얼↓이면 고유어·일상어. 종결만 바꾸고 어휘가
   따로 놀면 어색하다(예: 반말인데 "상기 사항을 숙지" → "위 내용 알아둬").
4. **호칭·지칭 일치** — 독자 지칭(귀하/고객님/여러분/너), 자기 지칭(저희/우리)을 목표 격식에 맞춤.
5. **구어 요소 조정** — 구어화면 문장 분할·담화 표지 추가, 문어화면 접속부사·관형절로 응집.
6. **장식 관습** — SNS·마케팅은 이모지·줄바꿈 허용, 공문·학술은 배제. 매체 관습을 따른다.

## 5. 보존 원칙 (Do-NOT)

문체는 바꾸되 다음은 보존한다.

- **사실·정보** — 수치·고유명사·날짜·인용·주장의 방향과 강도. 톤을 부드럽게 한다고 강도를 바꾸지 않는다.
- **핵심 메시지** — 글의 목적·요청·결론. 캐주얼하게 바꾼다고 핵심을 흐리지 않는다.
- **법적·계약 효력 문구** — 약관·고지·면책 등은 의미가 곧 효력이므로 격식 변환 시 특히 신중(필요시 표시).
- **인용·대사** — 따옴표 안 원화자의 말투는 함부로 바꾸지 않는다(화자 특성).

> 톤 변환은 **의미 보존 + register 변경**이다. 의미까지 바뀌면 그건 변환이 아니라 재작성이다(범위 밖).
